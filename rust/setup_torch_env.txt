PyTorch Environment Setup Commands
=====================================

The linker error is caused by incorrect library paths. Set these environment variables:

For Command Prompt (cmd):
-------------------------
set LIBTORCH=C:\libtorch
set LIBTORCH_LIB=C:\libtorch\lib
set LIBTORCH_INCLUDE=C:\libtorch\include
set LIBTORCH_BYPASS_VERSION_CHECK=1
set PATH=%PATH%;C:\libtorch\lib

For PowerShell:
---------------
$env:LIBTORCH = "C:\libtorch"
$env:LIBTORCH_LIB = "C:\libtorch\lib"
$env:LIBTORCH_INCLUDE = "C:\libtorch\include"
$env:LIBTORCH_BYPASS_VERSION_CHECK = "1"
$env:PATH += ";C:\libtorch\lib"

For Git Bash:
-------------
export LIBTORCH="C:/libtorch"
export LIBTORCH_LIB="C:/libtorch/lib"
export LIBTORCH_INCLUDE="C:/libtorch/include"
export LIBTORCH_BYPASS_VERSION_CHECK="1"
export PATH="$PATH:/c/libtorch/lib"

Then run:
---------
cargo build --features torch
cargo run --features torch

Alternative (Use the scripts):
------------------------------
run_torch.cmd        (for Command Prompt)
run_torch.ps1        (for PowerShell)

The Problem:
------------
The torch-sys build script was creating this incorrect path:
"/LIBPATH:C:\\libtorch\\lib\\lib"  (WRONG - double \lib)

Should be:
"/LIBPATH:C:\\libtorch\\lib"       (CORRECT)

The LIBTORCH_LIB environment variable fixes this by explicitly telling
the build script where to find the library files.
